# 문서 작성 가이드

**원칙**: 짧고, 명확하고, 실용적으로

---

## 기본 규칙

### 1. 길이 제한
- **최대**: 200줄 이내
- **목표**: 5분 안에 읽기
- **이유**: 아무도 긴 문서는 안 읽음

### 2. 구조
```
✅ 필수:
- 기본 규칙 (핵심만)
- 예시 1-2개
- 안티 패턴 1-2개

❌ 불필요:
- 장황한 설명
- 이론적 배경
- 중복된 예시
- 과도한 체크리스트
```

### 3. 작성 순서
1. 핵심 규칙만 적기
2. 예시 1개 추가
3. 끝
4. (정말 필요하면) 안티 패턴 1개

---

## 안티 패턴

### ❌ 과도한 설명
```markdown
## 왜 이 규칙이 필요한가?

역사적으로 살펴보면, 1990년대부터...
소프트웨어 공학의 관점에서...
학술적 연구에 따르면...

(3페이지 계속...)
```

**해결책**: 삭제. 바로 규칙만 적기.

### ❌ 중복된 예시
```markdown
## 예시

### 예시 1: 컴포넌트
...

### 예시 2: 컴포넌트 (거의 같음)
...

### 예시 3: 컴포넌트 (또 같음)
...
```

**해결책**: 예시 1개만 남기기.

### ❌ 과도한 체크리스트
```markdown
## 체크리스트

작성 전:
- [ ] ...
- [ ] ...
(20개 항목)

작성 중:
- [ ] ...
(20개 항목)

작성 후:
- [ ] ...
(20개 항목)
```

**해결책**: 핵심 3-5개만.

---

## 작성 후 검증

```bash
# 줄 수 확인
wc -l your-document.md

# 200줄 넘으면 절반으로 줄이기
```

**질문**:
- [ ] 5분 안에 읽을 수 있나?
- [ ] 핵심만 남았나?
- [ ] 불필요한 설명 없나?

## 좋은 예시

```markdown
# 짧은 제목

## 규칙
- 규칙 1
- 규칙 2

## 예시
[코드 1개]

## 안티 패턴
❌ 나쁜 예
✅ 좋은 예

끝.
```
